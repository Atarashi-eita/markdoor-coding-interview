# Mark Door コーディングテスト　Todo app


## 1. 概要

ユーザーがタスクを管理・追跡するためのシンプルなTODOアプリケーションです。
タスクの作成、表示、更新、削除などの基本操作をサポートし、直感的なUXを提供します。フロントエンドは最小限の構成とし、バックエンド（API）の設計と実装に重点を置いています。

## 2. 使用技術

* **バックエンド**: Python, FastAPI ,SQLAlchemy
* **データベース**: SQLite (`tasks.db` としてローカルに保存)
* **パッケージ管理**: uv
* **フロントエンド**: HTML, CSS, JavaScript (Vanilla JS)

## 3. 実装済み機能

要件定義に基づく以下の機能が実装されています。

* **タスクの追加**: タイトル（必須・30文字以内）と詳細説明（任意）を指定して作成できます。
* **タスクリストの表示**: 全タスクの一覧表示と、「すべて」「未完了」「完了済み」のフィルタリングが可能です。
* **タスクの編集**: タイトル、詳細説明、完了状態（未完了/完了）を更新できます。
* **タスクの削除**: 誤操作防止の確認プロンプトを経て、タスクを削除できます。
* **UI/UXフィードバック**: アクション後の即時反映、入力エラー時のアラート表示、完了済みタスクの取り消し線表示などに対応しています。

## 4. セットアップ手順

バックエンドAPIサーバーを起動した上で、フロントエンドの画面を開きます。

### 4-1. バックエンドの起動

**方法A: ローカル環境で起動する場合**
前提条件: パッケージ管理ツール `uv` がインストールされていること。

1. `backend` ディレクトリに移動します。
```bash
cd backend

```


2. 依存パッケージをインストールします。
```bash
uv sync

```


3. サーバーを起動します。
```bash
uv run uvicorn src.main:app --reload

```


APIサーバーが `http://127.0.0.1:8000` で起動します。

**方法B: Docker環境で起動する場合**

1. `backend` ディレクトリに移動し、イメージをビルド・起動します。
```bash
cd backend
docker build -t todo-backend .
docker run -p 8000:8000 todo-backend

```



### 4-2. フロントエンドの起動

バックエンドサーバーが起動している状態で、以下の手順を実行します。

1. ブラウザで `frontend/index.html` を直接開いてください。
（または、VSCodeのLive Serverなどのローカルサーバー機能を使用して開いてください）

## 5. 開発者向け情報

開発やコード修正を行う場合の手順です。

### 5-1. pre-commitの設定（初回のみ）

コードの品質（フォーマットやLint）を保つため、Gitのコミット時に自動チェックが走るように設定します。`backend` ディレクトリ内で以下を実行してください。

```bash
uv run pre-commit install

```

### 5-2. テストの実行

バックエンドの自動テスト（正常系・異常系）を実行します。

```bash
uv run pytest

```

## 6. 今後の課題・展望

本番運用や今後の拡張を見据え、以下の点について改善を予定しています。

* **データベース管理の堅牢化**
* `Alembic` を導入し、データベースのスキーマ変更（マイグレーション）を安全に管理する。
* FastAPIの `lifespan` イベントを導入し、アプリケーションの起動・終了時のDB初期化や接続管理をより適切に行う。


* **セキュリティ設定の最適化**
* 現在は開発用として全許可（`*`）となっているCORSミドルウェアの設定を見直し、許可するオリジンやメソッドを最小限に制限する。
